# Оценка вклада спектральных каналов в кластеризацию объектов

## Тема работы

Тема лабораторной работы: **оценка вклада спектральных каналов в кластеризацию объектов**.

Работа выполняется на датасете `SAT-4`, содержащем многоканальные изображения участков земной поверхности.

Ссылка на датасет: [SAT-4](https://www.kaggle.com/datasets/crawford/deepsat-sat4)

## Общая идея работы

Цель анализа состоит в том, чтобы определить, насколько разные спектральные каналы изображения помогают разделять объекты на естественные группы.

Вместо построения модели классификации основной акцент делается на:

- снижении размерности данных
- кластеризации объектов
- сравнении качества кластеризации для разных наборов каналов

Итоговый вопрос исследования:

> какие спектральные каналы дают наилучшее разделение объектов при кластеризации?

## Краткая информация о данных

Датасет `SAT-4` представляет собой набор изображений земной поверхности.

Структура локального датасета:

- `X_train_sat4.csv` — обучающая выборка признаков
- `X_test_sat4.csv` — тестовая выборка признаков
- `y_train_sat4.csv` — метки классов для train
- `y_test_sat4.csv` — метки классов для test
- `sat4annotations.csv` — расшифровка классов

Основные характеристики:

- всего объектов: `500000`
- train: `400000`
- test: `100000`
- один объект: `3136` признаков
- это соответствует изображению `28 x 28 x 4`
- каналы: `R`, `G`, `B`, `NIR`

Классы:

- `barren_land`
- `trees`
- `grassland`
- `none`

## Представление данных

Каждая строка в `X_*.csv` содержит `3136` чисел.

Экспериментально установлено, что строка корректно преобразуется в изображение через:

```python
reshape(28, 28, 4)
```

Это означает, что данные упорядочены по пикселям, а внутри каждого пикселя значения идут в порядке:

- `R`
- `G`
- `B`
- `NIR`

Следовательно, каналы можно выделять напрямую из CSV без восстановления изображения:

- `R` -> столбцы `0, 4, 8, ...`
- `G` -> столбцы `1, 5, 9, ...`
- `B` -> столбцы `2, 6, 10, ...`
- `NIR` -> столбцы `3, 7, 11, ...`

## Целевая задача анализа данных

Целевая задача работы:

> оценить вклад отдельных спектральных каналов и их комбинаций в качество кластеризации объектов земной поверхности.

Для этого будут сравниваться несколько вариантов представления данных:

- только `R`
- только `G`
- только `B`
- только `NIR`
- `RGB`
- `RGB + NIR`

## Почему выбрана именно эта задача

Данный датасет хорошо подходит для такого исследования, потому что:

- содержит большой объём данных
- объекты представлены в многоканальном виде
- имеются реальные классы, с которыми можно дополнительно сравнивать кластеры
- можно анализировать не только сами кластеры, но и вклад разных спектральных каналов

Такая постановка задачи соответствует дисциплине "Большие данные", поскольку позволяет исследовать структуру большого массива наблюдений и сравнить несколько вариантов обработки данных.

## Основной алгоритм работы

Для анализа предлагается использовать связку:

- `IncrementalPCA`
- `MiniBatchKMeans`

### 1. IncrementalPCA

`IncrementalPCA` — это алгоритм снижения размерности данных.

Он:

- уменьшает число признаков
- сохраняет основные направления изменчивости данных
- работает батчами
- подходит для больших наборов данных

В данной работе он нужен для:

- уменьшения размерности пространства признаков
- сокращения шума
- подготовки данных к кластеризации

### 2. MiniBatchKMeans

`MiniBatchKMeans` — это масштабируемая версия алгоритма `KMeans`.

Он:

- разбивает объекты на заданное число кластеров
- работает по частям, используя мини-батчи
- подходит для больших выборок

В данной работе он нужен для:

- построения кластеров объектов
- сравнения качества разделения данных при разных наборах каналов

## Почему выбраны именно эти алгоритмы

### Почему `IncrementalPCA`

- исходные данные имеют высокую размерность: `3136` признаков на объект
- обычная кластеризация в таком пространстве менее устойчива
- алгоритм позволяет обрабатывать данные батчами
- даёт компактное представление данных

### Почему `MiniBatchKMeans`

- хорошо масштабируется на больших данных
- проще и быстрее классического `KMeans` на крупных выборках
- позволяет явно задать число кластеров
- хорошо сочетается со снижением размерности через `IncrementalPCA`

### Почему не выбраны другие методы

- `DBSCAN` хуже масштабируется на таком объёме данных
- иерархическая кластеризация слишком тяжела по времени и памяти
- методы классификации не являются центральной задачей данной работы

## Методы и критерии оценки

Для оценки качества кластеризации предлагается использовать следующие метрики:

- `Silhouette Score`
- `Davies-Bouldin Index`
- `Calinski-Harabasz Index`

### 1. Silhouette Score

Показывает, насколько объект хорошо соответствует своему кластеру и насколько он удалён от соседнего кластера.

Интерпретация:

- значения ближе к `1` — лучше
- около `0` — кластеры пересекаются
- меньше `0` — плохое разбиение

Почему полезна:

- позволяет оценить качество разделения объектов
- хорошо подходит для сравнения разных наборов каналов

### 2. Davies-Bouldin Index

Оценивает качество кластеризации через сравнение:

- внутреннего разброса кластеров
- расстояний между кластерами

Интерпретация:

- чем меньше значение, тем лучше

Почему полезна:

- показывает, насколько кластеры компактны и отделены друг от друга
- хорошо дополняет `Silhouette Score`

### 3. Calinski-Harabasz Index

Сравнивает:

- межкластерный разброс
- внутрикластерный разброс

Интерпретация:

- чем больше значение, тем лучше

Почему полезна:

- показывает, насколько выражена кластерная структура данных
- позволяет сравнивать разные варианты признаков

## Почему эти метрики подходят для работы

Они полезны, потому что:

- позволяют оценивать кластеризацию без обучения классификатора
- подходят для сравнения нескольких наборов признаков
- помогают количественно оценить вклад каналов `R`, `G`, `B`, `NIR`
