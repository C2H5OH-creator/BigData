# BigData: XOR Arbiter PUFs

Репозиторий с лабораторной работой по предмету "Большие данные".  
Тема исследования: моделирование `XOR Arbiter PUF` с помощью логистической регрессии с `L1`-регуляризацией и анализ интерпретируемости признаков.

## Цель работы

Проверить, может ли линейная модель:
- выявить информативные признаки в данных `XOR Arbiter PUF`;
- выделить наиболее важные биты;
- показать устойчивый отбор признаков при изменении обучающей выборки;
- дать выигрыш при использовании `phi-features` по сравнению с `raw`-признаками.

Дополнительно в работе рассматриваются:
- зависимость времени обучения от объема данных;
- зависимость времени обучения от числа вычислителей.

## Датасет

В работе используются два поднабора:
- `6xor_64bit`
- `5xor_128bit`

Каждая строка содержит:
- входные биты `challenge`;
- целевой отклик `PUF`.

В ноутбуке рассматриваются два представления признаков:
- `raw` — исходные биты;
- `phi` — преобразованные признаки для проверки линейной моделируемости.

## Основные результаты

По итогам текущего анализа:
- качество модели оказалось близким к случайному:
  - `Accuracy ≈ 0.5`
  - `ROC-AUC ≈ 0.5`
  - `LogLoss ≈ 0.693`
- `L1`-регуляризация не выделила небольшой набор доминирующих признаков;
- коэффициенты модели не показывают выраженно важных битов;
- различия между `raw` и `phi-features` оказались несущественными;
- устойчивость отбора признаков с ростом выборки повышается, но остается ограниченной;
- увеличение числа вычислителей не дало заметного выигрыша по времени в выбранной постановке.

Итоговый вывод: выбранная линейная модель не выявила сильной и надежной структуры в данных `XOR Arbiter PUF`, что согласуется с высокой сложностью моделирования такого типа PUF.
