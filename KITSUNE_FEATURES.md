# Kitsune: описание уникальных признаков

В датасете Kitsune используется `115` колонок:

- `23` уникальных признака по смыслу
- каждый из них считается в `5` затухающих временных окнах

## Временные окна

Во всех группах используются одни и те же окна:

- `L5` -> примерно `100 ms`
- `L3` -> примерно `500 ms`
- `L1` -> примерно `1.5 s`
- `L0.1` -> примерно `10 s`
- `L0.01` -> примерно `1 min`

Это не обычные скользящие окна, а экспоненциально затухающие статистики.

## Что означают группы признаков

- `MI_dir_*`: трафик, исходящий от конкретной пары `SrcMAC + SrcIP`
- `H_*`: весь исходящий трафик от `SrcIP`, без учета получателя
- `HH_*`: трафик между двумя IP-адресами `SrcIP <-> DstIP`
- `HH_jit_*`: межпакетные задержки на направлении `SrcIP -> DstIP`
- `HpHp_*`: трафик между двумя сокетами `SrcIP:SrcPort <-> DstIP:DstPort`

Для `ARP` в `HpHp` код использует не сокеты, а пару `SrcMAC <-> DstMAC`.

## Что означают суффиксы статистик

- `weight`: эффективное число наблюдений в окне после экспоненциального затухания; по сути это затухающая интенсивность или "вес" потока
- `mean`: затухающее среднее значение
- `variance`: затухающая дисперсия
- `std`: стандартное отклонение
- `magnitude`: совместный масштаб двух потоков по средним значениям
- `radius`: совместный масштаб разброса двух потоков
- `covariance`: ковариация между двумя связанными потоками
- `pcc`: коэффициент корреляции Пирсона между двумя связанными потоками

Для `MI_dir`, `H`, `HH`, `HpHp` среднее и разброс считаются по размеру пакета.

Для `HH_jit` среднее и разброс считаются по межпакетным интервалам во времени.

## 23 уникальных признака по смыслу

### 1D-признаки по `SrcMAC + SrcIP`

- `MI_dir_weight`: насколько активно в окне отправляет трафик конкретная пара `MAC+IP`
- `MI_dir_mean`: средний размер пакета для этой пары `MAC+IP`
- `MI_dir_variance`: разброс размеров пакетов для этой пары `MAC+IP`

### 1D-признаки по `SrcIP`

- `H_weight`: насколько активно в окне отправляет трафик источник `SrcIP`
- `H_mean`: средний размер пакета от данного `SrcIP`
- `H_variance`: разброс размеров пакетов от данного `SrcIP`

### 2D-признаки по каналу `SrcIP <-> DstIP`

- `HH_weight`: активность канала между двумя IP в окне
- `HH_mean`: средний размер пакета на канале между двумя IP
- `HH_std`: разброс размеров пакетов на канале между двумя IP
- `HH_magnitude`: общий масштаб трафика пары IP по средним значениям
- `HH_radius`: общий масштаб разброса трафика пары IP
- `HH_covariance`: насколько изменения размеров пакетов у связанных потоков согласованы
- `HH_pcc`: нормированная корреляция между связанными потоками канала

### Jitter-признаки по направлению `SrcIP -> DstIP`

- `HH_jit_weight`: сколько межпакетных интервалов накоплено в окне
- `HH_jit_mean`: средний интервал между пакетами на направлении `SrcIP -> DstIP`
- `HH_jit_variance`: разброс межпакетных интервалов на направлении `SrcIP -> DstIP`

### 2D-признаки по сокету `SrcIP:SrcPort <-> DstIP:DstPort`

- `HpHp_weight`: активность сокетного канала в окне
- `HpHp_mean`: средний размер пакета в сокетном канале
- `HpHp_std`: разброс размеров пакетов в сокетном канале
- `HpHp_magnitude`: общий масштаб трафика пары сокетов по средним значениям
- `HpHp_radius`: общий масштаб разброса трафика пары сокетов
- `HpHp_covariance`: согласованность изменений размеров пакетов у пары сокетов
- `HpHp_pcc`: нормированная корреляция между потоками пары сокетов

## Как из 23 признаков получаются 115 колонок

Берется каждый из 23 признаков выше и повторяется для пяти окон:

- сначала для `L5`
- затем для `L3`
- затем для `L1`
- затем для `L0.1`
- затем для `L0.01`

Итого: `23 x 5 = 115`.

## Важная оговорка по коду репозитория

В текущем `Kitsune-py` есть расхождение между статьей и реализацией Python:

- статья и официальный датасет подразумевают отдельный блок `H_*`, то есть всего `115` признаков
- в текущем `netStat.py` блок `H` закомментирован, поэтому сам Python-код в таком виде фактически выдает только `100` признаков

Также в `AfterImage.py` 1D-статистика внутри `allstats_1D()` считает именно дисперсию, хотя в некоторых местах названия исторически используют `std`.

Поэтому для официального датасета корректно ориентироваться на схему `23 признака x 5 окон`, а не на текущее состояние Python-скрипта.
